# WEBHEADS

WEBHEADS — учебный веб-проект, разработанный в рамках дипломной работы.

Сайт предназначен для представления услуг IT-компании, публикации новостей, демонстрации портфолио и обработки заявок пользователей через форму обратной связи.

## Используемые технологии

- PHP
- MySQL
- PDO
- HTML5
- CSS3
- JavaScript
- Git / GitHub

## Конфигурация проекта

Для подключения проекта к базе данных необходимо создать файл:

```text
/include/config.php
```

Пример содержимого файла:

```php
<?php

return [
    'db_host' => 'localhost',
    'db_name' => 'database_name',
    'db_user' => 'database_user',
    'db_pass' => 'database_password',
];
```

Файл `config.php` содержит приватные данные доступа к базе данных и не должен загружаться в репозиторий.

В файл `.gitignore` необходимо добавить:

```gitignore
/include/config.php
/.idea/
/.vscode/
.DS_Store
```

## База данных

В проекте используется СУБД MySQL.  
Для подключения к базе данных применяется PDO.

При первом запуске проекта выполняется проверка наличия необходимых таблиц. Если таблицы отсутствуют, они создаются автоматически.

Основные таблицы базы данных:

- `feedback` — заявки пользователей
- `news` — новости сайта
- `portfolio` — проекты портфолио
- `services` — услуги компании

## Git Flow

Разработка проекта ведется через систему контроля версий Git с использованием отдельных веток под задачи.

Основная стабильная ветка проекта:

```text
main
```

Новая функциональность и доработки выполняются в отдельных ветках. Название ветки должно соответствовать типу изменения.

## Правила именования веток

### Задачи из трекера

Формат:

```text
task_9999
```

Где `9999` — номер задачи из трекера.

Пример:

```text
task_1024
```

### Срочные исправления вне задач

Формат:

```text
hot_fix
```

Пример:

```text
hot_fix
```

### Прочие доработки вне задач

Формат:

```text
feat
```

Пример:

```text
feat
```

## Правила именования коммитов

Коммиты должны иметь понятное описание выполненных изменений.

### Коммиты по задачам

Формат:

```text
task_9999 | Краткое описание задачи
```

Пример:

```text
task_1024 | Доработки главной страницы
```

### Срочные исправления

Формат:

```text
hot_fix | Краткое описание исправления
```

Пример:

```text
hot_fix | Исправил адрес
```

### Прочие доработки

Формат:

```text
feat | Краткое описание доработки
```

Пример:

```text
feat | Добавил форму обратной связи
```

## Порядок работы с Git

Получить актуальную версию проекта:

```bash
git pull origin main
```

Создать новую ветку под задачу:

```bash
git checkout -b task_1024
```

Добавить измененные файлы:

```bash
git add .
```

Создать коммит:

```bash
git commit -m "task_1024 | Доработки главной страницы"
```

Отправить ветку в удаленный репозиторий:

```bash
git push origin task_1024
```

После проверки изменений ветка объединяется с основной веткой `main`.

## Запуск проекта

1. Скопировать файлы проекта на локальный или удаленный сервер.
2. Создать базу данных MySQL.
3. Создать файл `/include/config.php`.
4. Указать данные подключения к базе данных.
5. Открыть сайт в браузере.

## Назначение проекта

Проект разработан для демонстрации навыков создания веб-ресурса с использованием PHP, MySQL, HTML, CSS и JavaScript. В рамках проекта реализована структура сайта IT-компании, подключение к базе данных, хранение заявок, отображение контентных разделов и базовая организация кодовой базы.
